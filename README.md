# diplom-chord-recognition

Chord-Recognition
Automatic chord recognition in Python
Chords are identified automatically from monophonic/polyphonic audio.

---------------------
О матрице отклонений:
---------------------
Матрица формируется по результатам тестирования папки аудиофайлов и представлена в csv и png формате.

PNG:

1) На оси X представлены истинные аккорды в формате "Аккорд (количество верно предсказанных файлов с этим аккордом / общее количество файлов с этим истинным аккордои)"
напр. C:maj (1/2)
2) На оси Y представлены предсказанные аккорды.
3) Каждому истинному аккорду принадлежит строка с совокупностью предсказанных аккордов и указанием их процентного соотношения по времени распознавания 
(в случае, если файлов истинного аккорда несколько, все проценты делятся на количество файлов).
4) Точность (% времени) - суммарный % времени из всех файлов, когда аккорд был угадан.
5) Точность (кол-во) - процент верно угаданных файлов (верным считается файл, в котором истинный аккорд был предугадан большее относительно остальных аккордов время).
6) Тишина (суммарно) - суммарное время из всех файлов, когда не было распознано ни одного аккорда (данная информация служит для установки корректного порога) 

----------------------------
Генерация матрицы отклонений:
----------------------------

python evaluate.py 

C алгоритмом EPCP вместо стандартного librosa:
python evaluate.py -c

С выбором порогов: 
python evaluate.py -t 0.3 0.5 0.6

Выбор тестовой папки осуществляется в начале функции main в строчке:
test_dir = "data/test_chords/base"

--------------------------------------
Запуск алгоритма для отдельного файла:
--------------------------------------

python main.py -i "G_open_strum.wav"

С графиками:
python main.py -i "G_open_strum.wav" -p

С алгоритмом EPCP вместо стандартного librosa:
python main.py -i "G_open_strum.wav" -p -c

С настройкой порога (дефолтный порог = 0.1):
python main.py -i "G_open_strum.wav" -p -t 0.3

Также в консоль выводится скорость работы алгоритма
